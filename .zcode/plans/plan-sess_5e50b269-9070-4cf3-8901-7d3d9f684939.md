## Plan: Detail View Fields + Smart Faceted Filters + Mobile Improvements

### Problem 1: Detail view missing all new fields

`openDetail()` renders vis-chips for pattern, style, density, corner, shadows — but completely omits **7 fields** that are in the data: `components`, `domainTags`, `colorScheme`, `mood`, `industryVertical`, `responsiveBehavior`, `accessibilityRisks`.

**Fix:** Add a new section to `openDetail` between the existing vis-chips and the critique — a "Classification" block showing:
- `colorScheme` + `mood` as eyebrow chips (light/dark, "confident and restrained")
- `domainTags` as pill chips
- `components` as pill chips (top 6, "+N more" if more)
- `industryVertical` + `responsiveBehavior` as vis-chips alongside density/corner
- `accessibilityRisks` as a red-tinted list under anti-patterns (with `[inferred]`/`[visible]` confidence tags)

### Problem 2: No smart/faceted filters — only single-select chips + text search

Current: mutually-exclusive chips (ALL / Starred / Exceptional / Cautionary / Mobile / Dark). Can't combine "dark + dashboard + fintech."

**Fix:** Replace the single `st.filter` string with a **multi-dimensional filter state** — each dimension is independently toggleable:
- `st.tier` — null | "exceptional" | "cautionary" (replaces tier chip)
- `st.platform` — null | "mobile" | "web" (replaces mobile chip)
- `st.colorScheme` — null | "light" | "dark" (replaces dark chip)
- `st.domainTag` — null | string (dropdown from corpus data)
- `st.industry` — null | string (dropdown from corpus data)
- `st.pattern` — null | string (dropdown from corpus data)

**UI:** Keep existing chips for tier (exceptional/cautionary are most common). Replace the single row of chips with two rows:
- **Row 1 (chips):** ALL, Exceptional, Cautionary, Starred — these are the quick toggles
- **Row 2 (smart dropdowns):** Pattern, Color Scheme, Industry, Domain — each is a `<select>` populated from the corpus data, with counts. "All" default. These compose with AND.

The `filtered()` function becomes: apply each active filter dimension as an `&&` predicate, then apply text search on top.

### Problem 3: Mobile gallery cards don't show enough context

On mobile (2-col at 560px), cards show: thumb + title + source + date + tier dot + score pips + palette strip. Missing: patternType label (currently just an icon overlay), mood, domainTags — the things that help you scan and decide what to tap.

**Fix:**
- Show `mood` as a small italic line under the title on cards (1 line, truncated)
- Show `colorScheme` as a small ●/○ indicator (dark/light) next to the tier dot
- Show top `domainTag` as a tiny chip below the palette strip
- Tighten the `.gfoot` row for narrow cards: score pips → just the number ("4/5")
- On the detail bottom-sheet: add a close affordance (tap the grab handle to close, not just the X)
